# 本ツール作成のために調査した事項のメモ

## 用紙サイズ

A4なので「210x297mm」

## 余白

上下左右ともに1インチ（25.4mm）

## 印刷領域

297 - 25.4 * 2 = 246.2mm

## ポイント・ピクセル・ミリメートルの対応

### ポイントとミリメートルの対応

- Excel の行の高さはポイントで指定する
- DTP や印刷では「1インチ = 72ポイント」が標準となっている
- 1ポイント = 1inch / 72pt = 25.4mm / 72pt≒ 0.35277777777778mm ≒ 0.35mm
- 1ミリメートル = 2.83464566929 ポイント ≒ 2.83 ポイント

### ピクセルとミリメートルの対応

- Windows の論理DPIは標準で 96dpi となっている
- 1インチ = 96ピクセル（Windowsの論理96dpi）
- 1ピクセル = 1inch / 96dpi = 25.4mm / 96dpi = 0.26458333333333mm ≒ 0.265mm
- 1ミリメートル = 96dpi / 25.4mm = 3.77952755905 ピクセル ≒ 3.78 ピクセル 
- 画面の解像度スケールを 125%・150% にすると DPI が変わるが、Excel は内部に独自の DPI (96dpi) を持っているため、実際は変化無し。

### ポイントとピクセルの対応

- 1pt = 96/72 px = 4/3 px ≈ 1.3333px
- 1px = 72/96 pt = 3/4 pt = 0.75pt

## `column.width` で指定した値と実際の幅の比較

`column.width` に指定した値と、出力された Excel ファイルを開いて確認したセルの幅の比較は下表のとおり。

| column.width | actual width | ratio       |
|--------------|--------------|-------------|
| 200px / 7    | 27.91pt      | 0.13955     |
| 300px / 7    | 42.18pt      | 0.1406      |
| 400px / 7    | 56.55pt      | 0.141375    |
| 500px / 7    | 70.82pt      | 0.14164     |
| 600px / 7    | 85.09pt      | 0.141816... |

比率が一定していないため、画像の幅から実際のセルの幅を求めるのは困難である。
